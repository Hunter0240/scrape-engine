Docker

A configurable web scraping engine in Python. Define what to scrape in YAML -- the engine handles fetching, parsing, cleaning, deduplication, storage, and notifications. No custom code per target.

Built with httpx (async HTTP), selectolax (C-backed HTML parser), and DuckDB (columnar storage with native Parquet export). Designed for freelance data collection work where each client gets a YAML config file, not a codebase. 

pip install -r requirements.txt
python -m engine run

Pipeline

Every scrape run passes through five stages. Each stage is a separate module that can be tested independently.

Fetch
HTTP request with rate limiting, retry, user-agent rotation
Parse
CSS selectors extract fields from each container element
Clean
Normalize whitespace, unicode, coerce types
Dedup
Hash key fields, skip records already in the database
Store
Insert into DuckDB, export CSV/JSON/Parquet, notify via webhook

Quick Start

# List configured targets
python -m engine targets

# Run all targets once
python -m engine run

# Run a specific target
python -m engine run targets/hackernews.yaml

# Run again -- dedup catches duplicates
python -m engine run
# -> hackernews: 30 scraped, 0 new, 30 duplicates

# Export stored data
python -m engine export hackernews -f csv
python -m engine export hackernews -f parquet

# Run on cron schedules (long-running)
python -m engine schedule

Target Config

Each target is a YAML file in the targets/ directory. This is the full schema:

FieldTypeDescription
name string required Unique identifier, used as the DuckDB table name
url string required Page URL to scrape
container string required CSS selector for the repeating element wrapping each record
fields map required Map of field name to selector config (see below)
rate_limit float Minimum seconds between requests (default: 1.0)
max_retries int Retry count for failed requests (default: 3)
key_fields list[string] Fields used for dedup hashing (default: all fields)
schedule string Cron expression for scheduled runs
webhook_url string Discord or Slack webhook for run notifications
headers map Extra HTTP headers to include in requests

Field Config

Each entry in fields defines how to extract one piece of data from the container element:

FieldTypeDescription
selector string required CSS selector relative to the container
attribute string Extract an HTML attribute (e.g. href) instead of text content
type string str, int, float, or url (default: str)

Example Targets

name: hackernews
url: https://news.ycombinator.com
container: "tr.athing"
rate_limit: 2.0
key_fields: [title, url]
schedule: "*/30 * * * *"

fields:
  rank:
    selector: "span.rank"
    type: int
  title:
    selector: "span.titleline > a"
  url:
    selector: "span.titleline > a"
    attribute: href
    type: url
  site:
    selector: "span.sitestr"
name: github-trending
url: https://github.com/trending
container: "article.Box-row"
rate_limit: 2.0
key_fields: [repo]
schedule: "0 */6 * * *"

fields:
  repo:
    selector: "h2 a"
    attribute: href
    type: url
  description:
    selector: "p"
  language:
    selector: "span[itemprop='programmingLanguage']"
  stars_today:
    selector: "span.d-inline-block.float-sm-right"

Pipeline Stages

Sends an async HTTP GET via httpx. Respects the target's rate_limit (minimum seconds between requests to the same target). Rotates through five real browser user-agent strings to avoid bot detection.

On network errors or HTTP 429 (rate limited), retries with exponential backoff: 1s, 2s, 4s. Gives up after max_retries attempts. HTTP errors other than 429 are raised immediately.

Uses selectolax (a C-backed HTML parser) for speed. Applies the container pattern: first find all elements matching the container selector, then extract each field relative to its container. This scopes selectors to individual records rather than the entire page.

If a field selector matches nothing inside a container, that field is set to null -- pages are messy and missing data shouldn't crash the pipeline.

Raw scraped text is often messy. The cleaner applies three transformations:

Unicode normalization (NFKC) converts compatibility characters like non-breaking spaces and full-width digits to their standard forms. Whitespace collapsing turns runs of spaces, tabs, and newlines into a single space. Type coercion converts strings to int or float based on the field's type config, stripping non-numeric characters first.

Scrapers run repeatedly and the same data shows up again. The deduper computes a SHA-256 hash from the target's key_fields (or all fields if not specified) and attaches it to each record as _hash.

The storage layer uses _hash as a primary key with INSERT OR IGNORE, so duplicate records are silently skipped at the database level. The run summary reports how many records were new vs. duplicates.

DuckDB is a columnar analytical database -- like SQLite but optimized for the kind of tabular data scraping produces. Tables are created automatically from the target's field definitions. Each record gets a _scraped_at timestamp.

Export to CSV, JSON, or Parquet via the CLI. Parquet export uses DuckDB's native COPY command for speed and produces files that load directly into pandas, Polars, or BigQuery.

CLI Reference

Run one or more targets immediately. With no arguments, runs all targets in targets/.

# Run all targets
python -m engine run

# Run specific targets
python -m engine run targets/hackernews.yaml targets/github-trending.yaml

Starts a long-running process that fires each target on its configured schedule (a cron expression). Targets without a schedule are skipped. The loop checks every 30 seconds. 

python -m engine schedule
# INFO: scheduler starting with 2 target(s):
#   hackernews (*/30 * * * *), github-trending (0 */6 * * *)

List all YAML target files with their URL, fields, and schedule.

python -m engine targets
# hackernews
#   url:      https://news.ycombinator.com
#   fields:   rank, title, url, site
#   schedule: */30 * * * *

Export one or more targets from DuckDB to a file. Supports CSV, JSON, and Parquet.

python -m engine export hackernews -f csv
python -m engine export hackernews github-trending -f parquet

Docker

The Docker image runs the scheduler by default. Mount data/ to persist the DuckDB database and targets/ to add or modify scrape targets without rebuilding. 

docker compose up --build
# docker-compose.yml
services:
  scraper:
    build: .
    env_file: .env
    volumes:
      - ./data:/app/data
      - ./targets:/app/targets
    restart: unless-stopped
